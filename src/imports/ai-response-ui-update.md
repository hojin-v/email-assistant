## 목표
수신함 우측 패널의 "AI 자동 생성 응답" 헤더와 관련 표현을
실제 서비스 방식(템플릿 매칭)에 맞게 수정한다.
변수 상태를 시각적으로 구분하는 표시 방식을 추가한다.

---

## 1. 패널 헤더 수정

### 변경 전
  아이콘: ✨ (스파클)
  제목: "AI 자동 생성 응답"
  부제목: "하이라이트된 필드를 클릭하여 내용을 수정하세요"

### 변경 후
  아이콘: 📄 (문서 아이콘, teal)
  제목: "매칭된 답변 템플릿"
  부제목: "하이라이트된 항목을 확인하고 필요 시 수정 후 발송하세요"

  헤더 아래 템플릿 출처 표시줄 추가:
    배경: 연한 회색 (#F3F4F6), 라운드 8px, 패딩 8px 12px
    좌측: 📄 아이콘 + "가격 안내 템플릿" (회색, 소형)
    우측: "템플릿 보기 →" (teal 텍스트 링크, 소형)

---

## 2. 변수 상태 표시 추가

### 패널 본문 상단에 변수 상태 요약 바 추가
  헤더와 받는 사람 필드 사이에 위치
  배경: 흰색, 하단 보더 라인 (연한 회색)
  
  3가지 상태 아이템을 가로로 나열:
    ✓ 자동 완성 4개   (teal 텍스트, 소형)
    👁 확인 필요 2개   (teal 텍스트, 소형)  
    ⚠ 입력 필요 1개   (amber 텍스트, 소형)
  
  각 아이템 사이 구분선 (|) 추가

### 본문 내 변수 하이라이트 스타일 정의

  자동 완성된 변수 (확실한 값):
    스타일 없음, 일반 텍스트로 표시
    예: "박민수님" → 그냥 텍스트

  AI/RAG가 채운 변수 (확인 권장):
    연한 teal 배경 (#E6FAF8)
    teal 텍스트
    클릭 시 수정 가능한 인풋으로 전환
    예: {{가격}} → 사용자당 월 49,000원

  미채운 변수 (입력 필요):
    연한 amber 배경 (#FEF3C7)
    amber 텍스트
    클릭 시 즉시 인풋 활성화
    예: {{담당자명}} → 클릭하여 입력

  각 하이라이트 변수에 마우스 오버 시:
    툴팁 표시
    확인 권장: "AI가 채운 값입니다. 확인 후 수정하세요"
    입력 필요: "직접 입력이 필요한 항목입니다"

---

## 3. 처리 상태별 패널 변형

### 검토 대기 상태 (현재 화면)
  위 변경사항 모두 적용
  하단 액션 버튼 유지:
    [발송하기] primary (teal)
    [편집 후 발송] secondary (outline)
    [건너뛰기] ghost

### 처리 완료 상태 (해당 이메일 선택 시)
  헤더:
    아이콘: 📄
    제목: "발송된 답변 내용"
    부제목 제거

  템플릿 출처 표시줄 유지

  변수 상태 요약 바 제거

  본문:
    모든 변수 하이라이트 제거, 일반 텍스트로 표시
    전체 텍스트 회색 처리 (읽기 전용 느낌)

  본문 상단 완료 배너 추가:
    배경: 연한 green (#DCFCE7)
    ✓ 처리 완료  ·  2025.03.03 오전 10:45 발송됨
    (green 텍스트, 소형)

  하단 액션 버튼 영역 제거

### 자동 발송 상태 (해당 이메일 선택 시)
  헤더:
    아이콘: ⚡
    제목: "자동 발송된 답변 내용"
    부제목 제거

  본문 상단 자동 발송 배너 추가:
    배경: 연한 green (#DCFCE7)
    ⚡ 자동 발송됨  ·  2025.03.03 오전 09:20
    (green 텍스트, 소형)

  본문 읽기 전용, 하단 버튼 영역 제거

---

## 디자인 유의사항

- 변수 하이라이트 색상은 반드시 구분되어야 함
  확인 권장(teal) vs 입력 필요(amber) 명확히 다르게
- 템플릿 출처 표시줄은 모든 상태에서 동일하게 유지
- 변수 상태 요약 바는 검토 대기 상태에서만 표시
- 한국어 텍스트 사용
- 기존 디자인 시스템 유지
