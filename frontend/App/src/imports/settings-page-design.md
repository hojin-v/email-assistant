## 목표
1. 사이드바에 설정 메뉴 추가 및 설정 페이지 신규 디자인
2. 프로필 드롭다운을 단순 조작만 남기도록 축소
3. 자동화 설정에서 이메일 연동 섹션 제거
4. 온보딩 및 비즈니스 프로필의 FAQ 입력 방식을
   Q/A 분리 구조로 변경

---

## 1. 사이드바 수정

### 메뉴 순서 변경
  대시보드
  수신함        (amber 숫자 배지 유지)
  캘린더
  템플릿 라이브러리
  비즈니스 프로필
  자동화 설정
  ───────────── (구분선)
  설정           ← 신규 추가
                 (톱니바퀴 라인 아이콘, 회색)

## 2. 설정 페이지 신규 디자인

### 페이지 헤더
  제목: "설정"
  부제목: "계정 및 서비스 환경을 관리합니다"

### 탭 구성 (페이지 상단 탭바)
  [계정]  [알림]  [화면]  [이메일 연동]
  활성 탭: teal 하단 밑줄 + teal 텍스트
  비활성 탭: 회색 텍스트

---

### 탭 1: 계정

  섹션 카드: "기본 정보"
    이름 입력 필드 (현재값: 김호진)
    이메일 입력 필드 (현재값: user@gmail.com)
    우측 하단: [저장] 버튼 (primary)

  섹션 카드: "비밀번호 변경"
    현재 비밀번호 입력 필드
    새 비밀번호 입력 필드
    새 비밀번호 확인 입력 필드
    우측 하단: [변경] 버튼 (primary)

---

### 탭 2: 알림

  섹션 카드: "알림 설정"
  각 항목은 좌측 설명 + 우측 토글 형태

    새 이메일 수신 알림
    "새 이메일이 도착하고 템플릿 매칭이 완료되면 알립니다"
    [토글 ON]

    검토 대기 초안 누적 알림
    "검토 대기 초안이 N개 이상 쌓이면 알립니다"
    [토글 ON]
    토글 ON 시 하단에 기준 건수 인풋 노출:
      "    개 이상일 때" (숫자 인풋, 기본값 3)

    이메일 계정 오류 알림
    "이메일 계정 연결이 끊기면 즉시 알립니다"
    [토글 ON]

    미분류 이메일 발생 알림
    "어떤 카테고리에도 매칭되지 않은 이메일이 생기면 알립니다"
    [토글 OFF]

    캘린더 등록 대기 알림
    "감지된 일정이 등록 대기 중일 때 알립니다"
    [토글 ON]

    자동 발송 일간 요약 알림
    "하루 자동 발송 결과를 오후 6시에 요약해서 알립니다"
    [토글 OFF]

  우측 하단: [저장] 버튼 (primary)

---

### 탭 3: 화면

  섹션 카드: "테마"
    라이트 / 다크 선택 카드 2개 (가로 나열)
    온보딩 어조 선택 카드와 동일한 스타일
    라이트 카드: 흰색 배경 미리보기 + "라이트" 텍스트
    다크 카드: 어두운 배경 미리보기 + "다크" 텍스트
    현재 선택: teal 테두리

  섹션 카드: "언어"
    드롭다운 (현재값: 한국어)

  섹션 카드: "대시보드 위젯 설정"
    부제목: "대시보드에 표시할 섹션과 순서를 설정합니다"
    드래그 가능한 항목 목록:
      ⠿  다가오는 일정      [표시 토글 ON]
      ⠿  최근 수신 이메일   [표시 토글 ON]
      ⠿  이번 주 요약       [표시 토글 OFF]
    (⠿ 아이콘은 드래그 핸들, 회색)
    항목 드래그로 순서 변경 가능한 UI

  우측 하단: [저장] 버튼 (primary)

---

### 탭 4: 이메일 연동

  기존 자동화 설정에 있던 이메일 계정 연동 내용을 이동
  캘린더 연동은 이 탭에 포함하지 않음

  섹션 카드: "연결된 이메일 계정"
    현재 연결된 계정 행:
      Gmail 로고 아이콘
      user@gmail.com
      연결 상태 배지: [정상 연결] (green)
      우측: [연결 해제] 버튼 (outline, red 텍스트)

    하단: [+ 계정 추가] 버튼 (dashed border 스타일)

  섹션 카드: "계정 추가" (+ 계정 추가 클릭 시 펼쳐짐)
    온보딩 이메일 연동 화면과 동일한
    Gmail / Microsoft Outlook / 기타 메일 카드 3개

## 3. 프로필 드롭다운 축소

### 변경 전 (기존)
  사용자 정보
  ─────────────
  계정: 이메일 계정 연동 관리 / 로그인 정보 변경 / 알림 설정
  화면: 대시보드 위젯 설정 / 언어 설정 / 테마 설정
  ─────────────
  로그아웃

### 변경 후 (축소)
  사용자 정보 영역:
    프로필 아바타 (이니셜 원형, 네이비)
    이름: "김호진" (굵게)
    역할 + 이메일: "관리자 · user@gmail.com" (회색, 소형)

  구분선

  설정 링크:
    톱니바퀴 아이콘 + "설정" → 설정 페이지로 이동

  테마 인라인 토글:
    달 아이콘 + "테마"
    우측: [라이트 / 다크] 텍스트 토글

  구분선

  로그아웃:
    로그아웃 아이콘 + "로그아웃" (red 텍스트)

  드롭다운 너비: 220px
  기존보다 간결하게

## 4. 자동화 설정 페이지 수정

  이메일 계정 연동 섹션 완전 제거
  해당 자리에 안내 텍스트 추가:
    회색 소형 텍스트:
    "이메일 계정 연동은 설정 > 이메일 연동에서 관리합니다 →"
    클릭 시 설정 페이지 이메일 연동 탭으로 이동

## 5. FAQ 입력 방식 변경

온보딩 비즈니스 자료 단계와 비즈니스 프로필 페이지
두 곳 모두 동일하게 적용

### 변경 전
  텍스트에어리어 1개에 자유 입력
  예시 텍스트: "Q: 환불 정책은?\nA: 14일 이내..."

### 변경 후
  각 FAQ 항목을 카드 형태로 분리

  FAQ 항목 카드 디자인:
    배경: 흰색, 라운드 8px, 회색 테두리 1px
    상단: "질문" 라벨 (회색 소형) + 텍스트 인풋
    하단: "답변" 라벨 (회색 소형) + 텍스트에어리어
    질문/답변 사이 구분선
    카드 우측 상단: [✕ 삭제] 아이콘 버튼 (회색, 호버 시 red)

  항목 추가 버튼:
    [+ 항목 추가] (dashed border 스타일, 전체 너비)
    클릭 시 새 FAQ 카드 하단에 추가

  항목이 없을 때 empty state:
    "FAQ 항목을 추가해주세요" (회색, 중앙 정렬)
    [+ 첫 항목 추가] 버튼

## 디자인 유의사항

- 설정 페이지는 기존 비즈니스 프로필, 자동화 설정과
  동일한 섹션 카드 레이아웃 스타일 적용
- 탭바는 설정 페이지 내부 상단에만 위치
  (전역 탭바 아님)
- FAQ 카드는 온보딩과 비즈니스 프로필 두 곳
  동일한 컴포넌트 스타일 적용
- 한국어 텍스트 사용
- 기존 네이비 사이드바, teal 액센트 디자인 시스템 유지
