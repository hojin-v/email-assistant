## 목표
수신함 화면을 전체 이메일 내역 확인이 가능하도록 재구성한다.
처리 대기 중인 이메일만 보이던 구조에서
전체 수신 이메일을 상태별로 필터링할 수 있는 구조로 변경한다.
매칭률 등 추상적 지표를 제거하고 처리 상태 중심으로 정보를 재편한다.
기존 3단 레이아웃(목록 / 원문 / 초안) 구조는 유지한다.

---

## 1. 수신함 목록 패널 (좌측) 수정

### 헤더 영역
  제목: "수신함" (기존 유지)
  부제목: "3개의 이메일 대기 중" → 제거

  헤더 아래 상태 필터 탭 추가:
    [전체] [대기 중 3] [처리 완료] [자동 발송됨]

    전체: 모든 수신 이메일
    대기 중: 검토 승인 대기 상태 (숫자 배지 표시)
    처리 완료: 사용자가 직접 검토 후 발송 완료
    자동 발송됨: 자동 발송 규칙으로 처리 완료

    활성 탭: teal 하단 밑줄 + teal 텍스트
    비활성 탭: 회색 텍스트
    숫자 배지: teal 원형 배지 (대기 중 탭에만 표시)

### 각 이메일 행 수정

  기존 구성:
    발신자명 / 시간 / 제목 / 카테고리 배지 / 매칭률% / 일정배지

  변경 후 구성:
    1행: 발신자명 (굵게) + 수신 시간 (우측, 회색)
    2행: 이메일 제목 (회색)
    3행: 카테고리 배지 + 처리 상태 배지

  처리 상태 배지 종류 (대시보드와 동일한 스타일):
    [검토 대기]   amber 배경, amber 텍스트
    [자동 발송]   green 배경, green 텍스트
    [처리 완료]   회색 배경, 회색 텍스트

  일정 감지 배지:
    기존 [🗓 일정] 배지 유지
    위치: 처리 상태 배지 우측

  매칭률(96%, 88% 등) 완전 제거

---

## 2. 이메일 원문 패널 (중앙) 수정

### 상단 헤더 영역 수정

  기존:
    [가격문의] [96% 매칭] [검토 후 발송]

  변경 후:
    [가격문의] [검토 대기]
    (매칭률 배지 제거, 처리 상태 배지로 교체)
    처리 상태 배지는 목록의 배지와 동일한 스타일 사용

### 원문 이메일 영역
  기존 디자인 유지

### 일정 감지 카드 수정

  기존 카드 구성 유지하되 버튼 영역만 수정:

  변경 전:
    [캘린더에 추가] [수정 후 추가] [✕]

  변경 후:
    [캘린더에 추가]  - primary (teal)
    [내용 수정 후 추가]  - secondary (outline)
    [이 이메일에서 무시]  - ghost 텍스트 버튼

  "내용 수정 후 추가" 클릭 시:
    카드가 인라인 편집 모드로 전환
    제목, 날짜/시간, 장소, 참석자 필드가 수정 가능한 인풋으로 변경
    하단: [캘린더에 추가] [취소] 버튼

---

## 3. AI 초안 패널 (우측) 수정

### 처리 완료 / 자동 발송된 이메일 선택 시 상태 표시

  처리 완료 이메일 선택 시:
    패널 상단에 완료 상태 배너 표시
    ✓ 처리 완료  2025.03.03 오전 10:45 발송됨  (green, 소형)
    초안 내용은 읽기 전용으로 표시 (편집 불가, 회색 처리)
    하단 버튼 영역 제거

  자동 발송된 이메일 선택 시:
    패널 상단에 자동 발송 상태 배너 표시
    ⚡ 자동 발송됨  2025.03.03 오전 09:20  (green, 소형)
    초안 내용 읽기 전용으로 표시
    하단 버튼 영역 제거

  검토 대기 이메일 선택 시:
    기존 편집 가능한 초안 패널 유지
    하단 액션 버튼:
      [발송하기] - primary (teal, 넓게)
      [편집 후 발송] - secondary (outline)
      [건너뛰기] - ghost 텍스트

### 초안 패널 상단 헤더
  기존: "AI 자동 생성 응답" + 설명 텍스트
  유지하되 처리 상태에 따라 헤더 텍스트 변경:
    대기 중: "AI 자동 생성 응답 — 하이라이트된 필드를 클릭하여 수정하세요"
    처리 완료: "발송된 응답 내용"
    자동 발송: "자동 발송된 응답 내용"

---

## 4. 사이드바 수신함 메뉴 수정

  기존: "수신함" 텍스트만 표시
  변경: "수신함" + 대기 중 숫자 배지 (amber 원형)
  대기 중 이메일이 없으면 배지 숨김

---

## 디자인 유의사항

- 3단 레이아웃 비율 유지 (목록 25% / 원문 40% / 초안 35%)
- 처리 상태 배지 색상은 대시보드와 완전히 동일하게 통일
  검토 대기: amber / 자동 발송: green / 처리 완료: 회색
- 매칭률 수치는 수신함 전체에서 완전히 제거
- 상태 필터 탭의 "대기 중" 탭이 기본 선택 상태로 표시
- 한국어 텍스트 사용
- 기존 네이비 사이드바, teal 액센트 디자인 시스템 유지
